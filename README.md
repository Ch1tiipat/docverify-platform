# 🛡️ DocVerify Platform

**DocVerify Platform** คือแพลตฟอร์มตรวจสอบความถูกต้องของเอกสารดิจิทัลระดับองค์กร (Enterprise Digital Document Verification Platform) ที่พัฒนาขึ้นมาด้วย **Next.js**, **Firebase** และ **EmailJS** ภายใต้แนวคิดความปลอดภัยของข้อมูลตามกฎหมายคุ้มครองข้อมูลส่วนบุคคล (**PDPA Compliant**) และใช้เทคโนโลยีการเข้ารหัสความปลอดภัยระดับทหาร (**SHA-256 Cryptographic Hashing**)

---

## 🌟 ฟีเจอร์หลักของระบบ (Key Features)

### 1. พอร์ทัลผู้ออกเอกสาร (Issuer Portal)
* **การออกเอกสารและเข้ารหัส:** กรอกข้อมูลเอกสารและอัปโหลดไฟล์จริงเพื่อคำนวณรหัสลับ SHA-256 (Hash) ผ่าน Web Crypto API ในเครื่องของผู้ใช้โดยตรง
* **การล็อกวันที่อิงเวลาโลก:** ฟิลด์วันที่ออกเอกสารจะดึงและล็อกวันที่ปัจจุบันของระบบโดยอัตโนมัติเพื่อป้องกันการสวมรอยแก้วันที่ออกเอกสารย้อนหลัง
* **ระบบปิดบังข้อมูล (Data Masking):** ป้องกันข้อมูลรั่วไหลโดยเซ็นเซอร์ข้อมูลก่อนบันทึกลงระบบฐานข้อมูล เช่น `John Smith` -> `J*** S***` และไม่จัดเก็บ Student ID ลงในฐานข้อมูลสาธารณะ
* **คิวอาร์โค้ดฝังโลโก้:** สร้างคิวอาร์โค้ดอัตโนมัติที่เป็นลิงก์วิ่งตรงเข้าสู่หน้าตรวจสอบ พร้อมฝังโลโก้ของระบบ (`docverify_logo.png`) ไว้ตรงกึ่งกลางแบบเจาะพิกเซลไม่ทับซ้อนกัน

### 2. ระบบส่งอีเมลอัตโนมัติ (Email Dispatch System)
* **การส่งเอกสารทางอีเมล:** เมื่อการออกเอกสารเสร็จสิ้น ระบบจะทำการยิงคำสั่งส่งอีเมลหาผู้รับ (ผู้ถือเอกสาร) ทันที
* **รายละเอียดในอีเมล:** ภายในอีเมลจะแนบรหัสเอกสาร, ชื่อเรื่อง, ค่า Hash, ลิงก์ดาวน์โหลดเอกสารต้นฉบับจาก Firebase Storage, ลิงก์สแกนตรวจสอบ และ**รูปภาพคิวอาร์โค้ด**ส่งไปให้ผู้รับเซฟเก็บไว้ทันที

### 3. หน้าประวัติการออกเอกสารแบบแยกส่วน (Document History Portal)
* **คลังตรวจสอบย้อนหลัง:** หน้า Dashboard พิเศษในเมนูข้างที่ให้ผู้ออกเอกสารเข้าถึงรายการใบรับรองที่ออกมาแล้วทั้งหมด (สูงสุด 100 รายการล่าสุด)
* **ระบบค้นหาแบบสด (Live Filter Search):** ค้นหารายการประวัติย่อยได้อย่างรวดเร็วเพียงแค่พิมพ์ รหัสเอกสาร, ชื่อเรื่อง, ชื่อผู้ถือ หรืออีเมล
* **ปุ่มดูรายละเอียดด่วน 👁️:** คลิกเปิดดูข้อมูลและคัดลอกรหัส Hash หรือเปิดรูป QR Code เพื่อส่งย้อนหลังให้ผู้รับได้ตลอดเวลาโดยไม่ต้องอัปโหลดทำใหม่

### 4. พอร์ทัลผู้ตรวจสอบเอกสาร (Verifier Portal)
* **การตรวจสอบสองช่องทาง:**
  1. **อัปโหลดไฟล์ตรวจสอบ:** ลากและวางไฟล์เอกสารเพื่อคำนวณ Hash และเปรียบเทียบในคลังข้อมูล
  2. **สแกนผ่านเว็บแคม:** เปิดกล้องบนคอมพิวเตอร์สแกนลิงก์ QR Code โดยระบบมีระบบแกะรหัสอัจฉริยะ (URL Parser) เพื่อสแกนลิงก์ Vercel ได้อย่างถูกต้อง
* **ความถูกต้องแม่นยำ:** ค้นหาประวัติการสแกนและบันทึกประวัติการตรวจสอบลงฐานข้อมูล (Audit Trail) แบบอัตโนมัติ

### 5. หน้าตรวจสอบอัตโนมัติสำหรับอุปกรณ์เคลื่อนที่ (Instant Verify Page)
* เมื่อนำมือถือเครื่องใดก็ได้มาสแกน QR Code ลิงก์จะนำทางผู้ตรวจสอบเข้าสู่หน้าเว็บย่อย `/verify?hash=[ค่า Hash]` โดยตรง
* ระบบจะทำการเปรียบเทียบข้อมูลฐานข้อมูลสดของ Firebase ตรวจเช็กวันหมดอายุการรับรอง และแสดงสถานะ **`VERIFIED`** (ถูกต้องแท้จริง) หรือ **`INVALID`** (ไม่ถูกต้อง) ทันทีโดยไม่ต้องกดผ่านขั้นตอนอื่น ๆ

## 🛠️ เทคโนโลยีที่เลือกใช้ (Tech Stack)

* **Frontend Framework:** Next.js 14+ (App Router)
* **Language:** TypeScript
* **Styling:** Tailwind CSS + shadcn/ui + Lucide Icons
* **Database & Storage:** Firebase Cloud Firestore & Firebase Storage
* **Email Service:** EmailJS API (REST endpoint)
* **Security:** Web Crypto API (SHA-256)
* **Libraries:** `html5-qrcode` (ระบบสแกนกล้อง), `qrcode.react` (ระบบเรนเดอร์คิวอาร์โค้ด)

## 🚀 ขั้นตอนการติดตั้งและรันระบบแบบพัฒนาในเครื่อง (Local Setup)

1. **โคลนและติดตั้งแพ็กเกจย่อย:**
   ```bash
   # แตกไฟล์ zip แล้วเปิดโฟลเดอร์ผ่าน VS Code หรือ Terminal
   npm install
   ```

2. **กำหนดค่าสภาพแวดล้อม (.env.local):**
   สร้างไฟล์ชื่อ `.env.local` ไว้ที่โฟลเดอร์นอกสุดของโปรเจกต์ และใส่ค่าเชื่อมโยงดังนี้:

3. **เริ่มระบบทดสอบ:**
   ```bash
   npm run dev
   ```
   เปิดบราว์เซอร์ไปที่ลิงก์: [http://localhost:3000](http://localhost:3000)

## 🌐 ขั้นตอนการอัปเดตและเผยแพร่ขึ้นบอร์ดจริง (Vercel Deployment)

เนื่องจากหน้าเว็บปกติจำเป็นต้องใช้อินเทอร์เน็ตในการสแกนคิวอาร์โค้ดข้ามอุปกรณ์ ระบบจึงออนไลน์อยู่บน Vercel:

1. **อัปโหลดโปรเจกต์ (ครั้งแรก):**
   ```bash
   npx vercel
   ```
   ทำตามขั้นตอนการล็อกอินและตั้งค่าเริ่มต้นบนหน้าจอคอมพิวเตอร์ของคุณ

2. **กำหนดค่า Environment Variables บนหน้าเว็บบอร์ด Vercel:**
   - เข้าไปที่หน้าแดชบอร์ดโครงการ Vercel ของคุณ -> ไปที่ **Settings** -> **Environment Variables**
   - คัดลอกค่าตัวแปรจากไฟล์ `.env.local` ทั้งหมดไปกด **Add** วางลงในตารางคีย์ข้อมูล

3. **อัปเดตขึ้นเว็บใช้งานจริง (Production Build):**
   หลังจากแก้ไขโค้ดเสร็จแล้ว ให้รันคำสั่งด้านล่างนี้ใน Command Prompt เสมอ:
   ```bash
   npx vercel --prod
   ```
   ระบบจะอัปเดตเว็บหน้าจริงของคุณที่: **`https://docverify-platform-main.vercel.app`**

## 🔒 แนวทางความปลอดภัยด้านข้อมูล (Data Security Policy)

1. **ความลับฝั่งผู้ใช้ (Client-side Security):** การคำนวณหาค่าลายเซ็นไฟล์ (File Hashing) ทำเสร็จสิ้นในคอมพิวเตอร์ของผู้ใช้ ไฟล์เอกสารจริงจึงไม่เสี่ยงต่อการรั่วไหลระหว่างคำนวณ
2. **PDPA Compliance:** ข้อมูลชื่อจริงของผู้รับจะถูกลบและแทนที่ด้วย Masking Pattern ในระบบหลังบ้าน เพื่อรับประกันความถูกต้องโดยไม่สร้างผลกระทบต่อสิทธิผู้ถือเอกสาร
3. **การป้องกันลิงก์สาธารณะ:** หน้าแสดงผลการตรวจรับรองภายนอก (`/verify`) จะแสดงผลเฉพาะข้อมูลที่ได้รับความยินยอมในการแสดง เช่น รหัสไอดี วันหมดอายุ และชื่อย่อ เพื่อความปลอดภัยสูงสุด
